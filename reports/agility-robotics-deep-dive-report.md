## 执行摘要

Agility Robotics是全球首家实现人形机器人商业化的公司，其Digit机器人已在GXO、Spanx等客户现场完成超10万个周转箱搬运任务。本报告从应用场景、技术方案、组织架构、商业模式四个维度，深度解析这家"务实派"人形机器人领军企业的成功密码。

**核心发现**：
- **场景聚焦战略**：专注仓储物流单一场景，不做通用机器人
- **RaaS商业模式**：首创Robot-as-a-Service，降低客户试用门槛
- **人机协作设计**：Digit专为与AMR、传送带配合而设计
- **务实技术路线**：专用AI+模块化硬件，强调可靠性而非炫技

## 第二章 应用场景深度解析

### 2.1 场景选择逻辑：为什么选仓储物流？

Agility选择仓储物流作为切入场景，基于以下考量：

| 维度 | 分析 |
|------|------|
| **市场需求** | 电商爆发，仓储人力短缺严重，美国仓储工人缺口超100万 |
| **任务特性** | 重复性高（搬箱子）、环境结构化（平坦地面、标准货架） |
| **技术可行** | 不需要复杂推理，定位+抓取即可，降低AI难度 |
| **付费意愿** | B端客户，ROI可量化，愿意尝试新技术 |
| **竞争空白** | AMR解决运输，但"最后一米"卸货/上货仍需人工 |

#### Digit工位详细描述

**工位名称**：AMR-to-Conveyor Transfer Station（AMR到传送带转运站）

**工位功能**：
1. **接收**：从AMR接收载有产品的周转箱（Tote）
2. **转运**：将周转箱放置到传送带上
3. **堆叠**：将周转箱堆叠到不同楼层位置

**工作流程**：
```
Step 1: AMR导航至Digit工作站
    ↓
Step 2: Digit识别AMR到达，定位周转箱位置
    ↓
Step 3: Digit伸手抓取周转箱（使用末端执行器）
    ↓
Step 4: Digit转身，将周转箱放置到传送带
    ↓
Step 5: 传送带将周转箱运送至包装站
    ↓
Step 6: Digit等待下一辆AMR
```

**工作参数**：
| 参数 | 数值 |
|------|------|
| 单次搬运重量 | 最高23kg（新一代Digit） |
| 周转箱尺寸 | 标准600×400×300mm（可定制） |
| 工作节拍 | 约30-60秒/箱（取决于距离） |
| 工作时间 | 24/7连续作业（换班充电） |
| 日处理量 | 约500-1000箱/台（估算） |

### 2.3 产品线：Digit的多场景适配

#### 场景1：AMR-to-Conveyor（AMR到传送带）
- **客户**：GXO Spanx工厂、亚马逊仓库
- **任务**：从AMR卸货，放置到传送带
- **价值**：填补AMR和传送带之间的"gap"，原需人工完成

#### 场景2：Truck Unloading（货车卸货）
- **客户**：试点中
- **任务**：从货车/集装箱内卸下货箱，放置到传送带
- **挑战**：狭窄空间、不规则堆放、各种尺寸箱子
- **技术**：基于吸盘的末端执行器+全身控制

#### 场景3：Tote Stacking（周转箱堆叠）
- **客户**：GXO（升级中）
- **任务**：将周转箱堆叠到不同高度、不同位置
- **能力展示**：多任务处理、空间推理

#### 场景4：Goods-to-Person（货到人辅助）
- **规划**：与AMR深度协作
- **任务**：AMR将货架运到Digit，Digit抓取特定SKU放置到订单箱
- **复杂性**：需要识别不同SKU，精准抓取

### 2.4 人机协作架构

```
【仓库自动化层级】

Level 5: WMS (仓库管理系统)
    ↓ 任务分配
Level 4: Agility Arc (Agility云平台)
    ↓ 机器人调度
Level 3: Digit (人形机器人)
    ↓ 物理操作
Level 2: AMR (自主移动机器人)
    ↓ 货物运输
Level 1: Conveyor (传送带)
    ↓ 固定路径输送
Level 0: Human (人类工人)
    ↓ 处理异常、维护
```

**协作关系**：
- Digit **调用** AMR：通过Arc平台发送指令，让AMR到达指定位置
- Digit **配合** 传送带：识别传送带状态，适时放置周转箱
- Digit **接受** WMS指令：获取任务清单（从哪取、放哪）
- Digit **求助** 人类：遇到无法处理的异常时暂停并告警

### 3.2 核心技术详解

#### 3.2.1 行走与平衡：从Cassie继承的底层能力

**鸵鸟腿设计**：
```
传统人形机器人：    Digit设计：
    髋关节 ──→ 膝关节 ──→ 踝关节
       ↑          ↑          ↑
    电机直驱    电机直驱    电机直驱
       │          │          │
    小腿垂直    小腿垂直    脚板平直

Digit创新：
    髋关节 ──→ 膝关节(向后弯) ──→ 踝关节
       ↑              ↑              ↑
    大扭矩电机     四连杆机构      弹性脚板
       │              │              │
    提供动力      储能+释放       吸收冲击
```

**技术原理**：
- **膝关节向后弯曲**：类似鸵鸟/鸟类，行走时重心波动更小
- **四连杆弹簧机构**：像跷跷板一样储能，每步节省30%+能量
- **被动动态行走**：利用惯性+重力，减少主动控制需求

**性能指标**：
- 行走速度：1.5m/s（人类正常步速）
- 能耗：比传统设计低40%
- 稳定性：可承受20kg侧向推力不倒

#### 3.2.2 感知与定位

**传感器配置**：
| 传感器 | 位置 | 功能 | 规格 |
|--------|------|------|------|
| RGB摄像头 | 头部×2 | 立体视觉、物体识别 | 1080p, 60fps |
| LiDAR | 胸上部 | 360°环境建图 | 16线/32线可选 |
| 深度相机 | 手臂×2 | 抓取点定位 | Intel RealSense类 |
| IMU | 躯干 | 姿态估计 | 9轴 |
| 编码器 | 各关节 | 位置/速度反馈 | 绝对值+增量式 |
| 力传感器 | 足部 | 地面接触检测 | 6轴力/力矩 |

**定位算法**：
```
输入：LiDAR点云 + 摄像头图像 + IMU数据
    ↓
SLAM算法（Cartographer/自定义）
    ↓
生成：实时地图 + 机器人位姿
    ↓
与AMR协同定位（通过Arc平台共享位置）
```

#### 3.2.3 抓取与操作

**末端执行器演进**：
```
Digit V1-V2：    简易夹爪（2指）
    ↓           可抓取规则物体
Digit V3：      升级夹爪（增加力控）
    ↓           抓取更稳定，不怕滑落
Digit V4：      多功能末端（快换接口）
    ↓           适配夹爪、吸盘等工具
新一代Digit：   智能末端（集成传感器）
    ↓           触觉反馈，抓取更精准
```

**抓取流程**：
```
Step 1: 视觉检测
    - 检测周转箱位置（3D边界框）
    - 识别箱把手/可抓取区域
    
Step 2: 抓取规划
    - 计算最佳抓取姿态
    - 避障检查（检查手臂路径）
    
Step 3: 接近与抓取
    - 手臂伸展，末端接近目标
    - 闭合夹爪/启动吸盘
    - 力传感器确认抓取成功
    
Step 4: 搬运与放置
    - 规划放置位置（传送带/货架）
    - 移动手臂，保持稳定
    - 松开夹爪/关闭吸盘
```

**抓取成功率**：
- 标准周转箱（有把手）：>99%
- 不规则箱子：>95%
-  Slip检测+重试机制：失败率<0.1%

#### 3.2.4 任务专用AI（vs 通用大模型）

Agility选择了**专用AI路线**，而非Figure AI的通用VLA大模型：

| 对比维度 | Agility专用AI | Figure AI通用VLA |
|----------|---------------|------------------|
| **训练数据** | 仓库场景专用数据 | 互联网+机器人数据 |
| **模型架构** | 模块化（检测+规划+控制） | 端到端Transformer |
| **推理速度** | 快（10-50ms） | 较慢（100-500ms） |
| **泛化能力** | 弱（只做搬运） | 强（听懂自然语言） |
| **可靠性** | 高（99.9%+成功率） | 中等（还在优化） |
| **开发成本** | 低（专注单一任务） | 高（需要大量数据） |

**Agility的AI架构**：
```
输入：摄像头图像 + LiDAR点云 + 任务指令
    ↓
【视觉模块】
├─ 物体检测：YOLOv8定制版，检测周转箱/AMR
├─ 位姿估计：PointNet++，计算3D位置和角度
└─ 状态识别：判断AMR是否到位、传送带是否运行
    ↓
【规划模块】
├─ 路径规划：RRT*，规划手臂运动轨迹
├─ 抓取规划：几何分析+神经网络，选择抓取点
└─ 全身控制：二次规划，优化质心轨迹
    ↓
输出：关节目标位置
    ↓
【控制模块】PID+前馈，1000Hz实时控制
```

## 第五章 商业模式与财务分析

### 5.1 RaaS（Robot-as-a-Service）模式详解

#### 5.1.1 什么是RaaS？

传统模式 vs RaaS模式：
```
【传统购买模式】
客户：一次性支付$250,000购买Digit
      ↓
      自购后自行维护、编程、升级
      ↓
      风险：技术迭代快，可能3年后过时

【RaaS租赁模式】
客户：按月/按任务支付费用
      ↓
      Agility提供：机器人+软件+维护+升级
      ↓
      优势：零首付、随时升级、风险共担
```

#### 5.1.2 RaaS定价策略

| 收费模式 | 价格 | 适用客户 |
|----------|------|---------|
| **按时长租赁** | $10,000-$15,000/月/台 | 长期稳定需求 |
| **按任务计费** | $0.5-$1/箱（搬运） | 波动需求 |
| **混合模式** | 基础月费+超额任务费 | 大多数客户 |

**成本对比**（以美国仓储工人为例）：
```
人工成本：
- 时薪：$15-20
- 每天8小时 × 250天 = 2,000小时/年
- 年薪：$30,000-$40,000
- 社保福利：+30% = $39,000-$52,000
- 5年总成本：$195,000-$260,000

Digit RaaS成本：
- 月费：$10,000
- 年费：$120,000
- 5年总成本：$600,000

【当前】Digit更贵，但优势：
- 24/7工作（3班倒人工需$600,000+）
- 不会疲劳、不会受伤
- 可远程监控、数据分析

【未来】成本下降后：
- Digit售价降至$100,000
- RaaS月费降至$5,000
- 5年成本$300,000，开始具备竞争力
```

#### 5.1.3 Agility Arc云平台

Arc是RaaS模式的核心支撑：
```
【Arc功能模块】

1. 任务编排（Fleet Management）
   - 从客户WMS接收任务
   - 分配给多台Digit协调执行
   - 优化路径，避免碰撞

2. 实时监控（Real-time Monitoring）
   - 查看每台Digit状态（在线/离线/忙碌）
   - 视频监控（可选）
   - 性能统计（处理量、成功率、效率）

3. 远程运维（Remote Maintenance）
   - 远程诊断故障
   - OTA软件更新
   - 预测性维护（提前更换易损件）

4. 数据分析（Analytics）
   - ROI计算（节省多少人工成本）
   - 效率优化建议
   - 故障模式分析

5. AMR集成（Fleet Integration）
   - 调度AMR到达Digit工作站
   - 与第三方AMR（Locus、6 River等）对接
```

### 6.2 竞争优势分析

- **唯一商业化运营**：真实场景24/7运行超2年

- **标准化产品**：针对不同仓库，80%配置相同，20%定制

#### 优势3：生态整合能力
- **与AMR协作**：不是替代AMR，而是补全"最后一米"
- **与WMS对接**：标准API，快速集成
- **与人力协作**：设计为辅助人工，而非完全替代
