## 一、核心贡献概述

Helix 是 Figure AI 推出的**首个通用型人形机器人视觉-语言-动作（VLA）模型**，实现了多项行业第一：

### 1.1 五大技术突破

| 突破 | 描述 | 行业意义 |
|-----|------|---------|
| **全上肢控制** | 首个输出高频率连续控制整个人形上半身的VLA，包括手腕、躯干、头部和单根手指 | 解决了高维动作空间控制的难题 |
| **多机协作** | 首个同时在两个机器人上运行的VLA，实现共享长程操作任务 | 开辟了多机器人协同新范式 |
| **万物抓取** | 可拾取数千种从未见过的家用物品，仅需自然语言指令 | 实现了真正的零样本泛化 |
| **单一神经网络** | 使用单组神经网络权重学习所有行为，无需任务特定微调 | 大幅简化部署和训练流程 |
| **商业就绪** | 首个完全在嵌入式低功耗GPU上运行的VLA | 具备即时商业部署能力 |

## 二、双系统架构：System 1 + System 2

Helix 的核心创新在于借鉴认知科学的**双系统理论**，将VLA解耦为两个互补系统：

```
┌─────────────────────────────────────────────────────────────┐
│                     Helix Architecture                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌──────────────┐         Latent Vector         ┌──────┐  │
│   │  System 2    │ ─────────────────────────────> │System│  │
│   │  (S2)        │    High-level intent          │  1   │  │
│   │              │    7-9 Hz                     │ (S1) │  │
│   │  • VLM (7B)  │                               │      │  │
│   │  • Scene     │                               │ 80M  │  │
│   │    understanding                             │transformer│
│   │  • Language  │                               │      │  │
│   │    comprehension                             │200 Hz│  │
│   └──────────────┘                               └──────┘  │
│          ↑                                          │      │
│          │                                          ↓      │
│   Camera + State                            Robot Actions   │
│   (RGB + proprioception)                    (35-DoF)       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.1 System 2 (S2)：慢思考系统

- **模型**: 7B 参数开源VLM（基于互联网预训练）
- **频率**: 7-9 Hz
- **输入**: 单目机器人图像 + 机器人状态（手腕姿态 + 手指位置）
- **输出**: 连续潜向量（编码高层行为意图）
- **功能**: 
  - 场景理解
  - 语言理解
  - 跨物体和场景的广泛泛化

### 2.2 System 1 (S1)：快思考系统

- **模型**: 80M 参数交叉注意力编码-解码 Transformer
- **频率**: 200 Hz
- **输入**: 
  - 与S2相同的图像和状态（更高频率）
  - S2提供的潜向量条件
- **输出**: 35自由度完整上半身控制
  - 期望手腕姿态
  - 手指屈曲和外展控制
  - 躯干和头部方向目标
  - 任务完成百分比（用于行为链式组合）

### 2.3 架构优势分析

| 维度 | 传统单系统VLA | Helix双系统 |
|-----|-------------|------------|
| **速度vs泛化** | 需要权衡 | 两者兼得 |
| **推理频率** | 统一频率（通常10-20Hz） | S1: 200Hz, S2: 7-9Hz |
| **模型复杂度** | 单一大模型 | 解耦优化，可独立迭代 |
| **动作空间** | 低维（如二值夹爪） | 高维连续（35-DoF） |
| **部署灵活性** | 受限 | 模型并行，异步执行 |

## 三、技术细节深度解析

### 3.1 数据收集与自动标注

**数据规模**: ~500小时高质量遥操作数据

**关键创新 - 自动语言标注**:
```
训练数据生成流程:
1. 遥操作收集机器人行为视频
2. VLM自动标注: "What instruction would you have given 
   the robot to get the action seen in this video?"
3. 生成 hindsight instruction 作为训练语言条件
4. 确保测试物品在训练中从未出现（零样本验证）
```

**数据效率**:
- 仅为先前VLA数据集的 <5%
- 与单任务模仿学习数据集规模相当
- 但实现了全上肢人形控制

### 3.2 视觉骨干网络

**S1的视觉处理**:
- 全卷积多尺度视觉骨干
- 完全在仿真中预训练
- 处理高频率视觉输入以支持闭环控制

**优势**:
- 仿真到现实的视觉特征迁移
- 多尺度特征捕获不同粒度的空间信息
- 全卷积设计支持不同分辨率输入

### 3.3 训练策略

**端到端训练**:
```
损失函数: L = ||a_pred - a_gt||²  (标准回归损失)

梯度流:
S1动作预测误差 → 回传至S1参数
                    ↓
              通过潜向量条件
                    ↓
              回传至S2参数
                    ↓
              联合优化双系统
```

**关键训练技巧 - 时间偏移**:
- 在S1和S2输入间引入时间偏移
- 偏移量匹配部署时的推理延迟差距
- 确保训练-部署分布一致性

**无任务特定适应**:
- 单阶段训练
- 单组网络权重
- 无单独动作头
- 无需每任务微调

### 3.4 部署优化

**模型并行推理**:
```
机器人硬件配置:
• 双嵌入式低功耗GPU

部署架构:
GPU 1 (S2): 
  - 异步后台进程
  - 消耗最新观测和语言指令
  - 持续更新共享内存中的潜向量

GPU 2 (S1):
  - 实时进程
  - 维持200Hz控制循环
  - 消费最新观测 + 最新S2潜向量
```

**异步执行优势**:
1. S1在观测上具有更高时间分辨率
2. 更紧的反馈循环实现反应性控制
3. 两系统在最优频率运行
4. 速度匹配最快单任务模仿学习策略

## 四、实验结果与能力展示

### 4.1 全上肢协调控制

**控制维度**: 35-DoF 动作空间 @ 200Hz

| 控制组件 | 挑战 | Helix解决方案 |
|---------|-----|--------------|
| **头部控制** | 移动改变视野，影响感知 | 与手部运动协调，保持目标跟踪 |
| **躯干控制** | 改变可达范围，影响平衡 | 优化姿态以最大化操作空间 |
| **手指控制** | 高维精细操作 | 精确控制屈曲和外展 |
| **反馈循环** | 头/躯干运动创造感知-动作耦合 | 200Hz高频控制维持稳定性 |

**历史意义**: 此前在如此高维动作空间实现这种精度被认为极其困难，即使是单任务。

### 4.2 零样本多机协作

**实验设置**:
- 两个Figure机器人协作完成杂货存储
- 使用**相同Helix模型权重**
- 操作**完全新颖**的物品（训练中从未出现）

**协调机制**:
```
自然语言协调示例:

机器人A指令: "Hand the bag of cookies to the robot on your right"
机器人B指令: "Receive the bag of cookies from the robot on your left 
              and place it in the open drawer"

结果: 灵活、扩展的协作操作，无需:
  - 机器人特定训练
  - 显式角色分配
  - 预编程协调协议
```

**行业首创**: 首个使用VLA实现多机器人灵活协作的演示。

### 4.3 涌现的"万物抓取"能力

**测试规模**: 数千种新颖家用物品

**物品种类**:
- 玻璃器皿
- 玩具
- 工具
- 衣物
- 各种形状、大小、颜色和材质

**抽象概念理解**:
```
指令: "Pick up the desert item"

Helix执行:
1. 语言理解: "desert item" → 仙人掌玩具（语义关联）
2. 视觉识别: 定位仙人掌
3. 动作规划: 选择最近的手
4. 精确控制: 执行抓取所需的运动命令
```

**能力来源**: 互联网规模语言理解与精确机器人控制的桥接。

### 4.4 长期任务执行

Helix可执行多步骤长程任务：

| 任务类型 | 行为链示例 |
|---------|-----------|
| **容器操作** | 放置物品到各种容器（抽屉、冰箱等） |
| **多步骤交互** | 打开抽屉 → 放置物品 → 关闭抽屉 |
| **协作任务** | 接收物品 → 判断放置位置 → 执行放置 |

**任务完成信号**: 合成的"percentage task completion"动作使Helix能预测自己的终止条件，便于行为链式组合。

## 五、技术创新点评

### 5.1 核心创新对比

| 创新点 | Helix方法 | 传统方法 | 优势 |
|-------|----------|---------|-----|
| **架构设计** | System 1 + System 2 解耦 | 单系统统一处理 | 速度与泛化兼得 |
| **动作空间** | 35-DoF 高维连续 | 低维离散（如7-DoF臂+二值夹爪） | 更自然的人形控制 |
| **控制频率** | 200 Hz 高频 | 10-20 Hz | 更流畅、稳定 |
| **多机部署** | 相同权重，语言协调 | 每机器人独立模型 | 更灵活、可扩展 |
| **数据效率** | ~500小时 | 通常>10,000小时 | 训练成本更低 |

### 5.2 与相关工作的关系

**对比 OpenVLA (Kim et al., 2024)**:
- OpenVLA: 开源VLA，主要面向机械臂
- Helix: 专为人形设计，全上肢控制

**对比 π0 (Physical Intelligence, 2024)**:
- π0: 基于扩散的VLA，通用机器人控制
- Helix: 双系统架构，专为人形优化

**对比 RT-2 (Google DeepMind, 2023)**:
- RT-2: 端到端VLA，动作token化
- Helix: 连续动作输出，避免tokenization复杂度

**对比 Diffusion Policy (Columbia, 2023)**:
- Diffusion Policy: 基于扩散的模仿学习
- Helix: VLA架构，整合视觉-语言-动作

### 5.3 关键技术洞察

1. **高维动作空间的VLA是可行的**
   - 挑战了"VLA只能处理低维控制"的假设
   - 双系统架构是关键使能技术

2. **语言作为协调机制**
   - 多机协作无需显式通信协议
   - 自然语言足以实现灵活协调

3. **数据效率突破**
   - 证明了高质量数据 > 大数据量
   - 自动标注减少人工成本

4. **仿真预训练的价值**
   - S1视觉骨干完全仿真预训练
   - 有效的sim-to-real迁移

3. **强化学习增强**
   - 当前仅模仿学习
   - 添加RL提升鲁棒性和最优性

4. **多模态指令**
