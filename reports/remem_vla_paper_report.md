# ReMem-VLA: 通过双层循环查询为VLA模型赋能记忆能力

**论文信息**: arXiv:2603.12942  
**标题**: Empowering Vision-Language-Action Model with Memory via Dual-Level Recurrent Queries  
**作者**: Hang Li, Ziyuan Liu, Fengyi Shen, Dong Chen, Liudi Yang, Xudong Wang, Jinkui Shi, Zhenshan Bing, Alois Knoll  
**机构**: 慕尼黑工业大学、华为海森堡研究中心、弗莱堡大学、南京大学、华为技术有限公司  

---

## 📋 摘要

用于闭环机器人控制的视觉-语言-动作(VLA)模型通常基于**马尔可夫假设**，这使得它们在需要历史上下文的任务上容易出错。为了融入记忆能力，现有VLA模型要么从记忆库中检索（容易被干扰物误导），要么扩展帧窗口（固定视界仍然限制长期记忆保持）。

本文提出 **ReMem-VLA**，一种配备两组可学习查询的循环记忆VLA模型：

1. **帧级循环记忆查询**: 在连续帧之间传播信息，支持**短期记忆**
2. **块级循环记忆查询**: 跨时间块携带上下文，支持**长期记忆**

这些查询通过端到端训练来聚合和保持时间相关的上下文，隐式指导模型决策，无需额外的训练或推理成本。此外，本文引入**过去观察预测**作为辅助训练目标来增强视觉记忆。

通过广泛的以记忆为中心的模拟和真实世界机器人实验，ReMem-VLA在多种记忆维度（空间、顺序、情景、时间、视觉）上表现出强大的记忆能力，显著超越无记忆VLA基线π₀.₅和OpenVLA-OFT，并在记忆依赖任务上大幅超越MemoryVLA。

## 🎯 研究背景与动机

### VLA模型的现状与局限

Vision-Language-Action (VLA) 模型已成为机器人领域的核心研究焦点。通过利用预训练的视觉-语言模型(VLM)，近期VLA在操作任务上取得了 promising 的性能并展示了强大的泛化能力。

**当前VLA的典型范式**:
- 基于当前或短期历史观察
- 结合语言指令预测未来动作块
- 实现闭环控制

**核心问题**: 这种**马尔可夫式范式忽略了时间因果性**，使机器人对其已完成的工作和环境的变化"视而不见"，导致在记忆依赖的操作任务上出现系统性失败。

### 记忆依赖任务示例

> 如果机器人被要求将杯子放回原始位置，VLA必须回忆杯子最初在哪里。

**VLA所需的五种关键记忆能力**（如图1所示）：

| 记忆类型 | 描述 | 示例 |
|----------|------|------|
| **顺序记忆** | 按正确顺序执行多步任务 | 先拿杯子再倒水 |
| **视觉记忆** | 使用过去的视觉线索指导当前决策 | 回忆物体最初的颜色/形状 |
| **空间记忆** | 保持物体位置信息 | 记住杯子最初放在哪 |
| **情景记忆** | 跟踪已执行的动作 | 已经按了几次按钮 |
| **时间记忆** | 表示持续时间和经过时间 | 浇水持续了多久 |

这些记忆跨越从**短期保持**（如保持固定姿势几秒钟）到**长期记忆**（如跟踪整体任务进度）。

### 现有记忆方法的局限

| 方法类型 | 代表工作 | 局限性 |
|----------|----------|--------|
| **记忆库检索** | MemoryVLA, MemER | 依赖强线索，大规模时易受干扰 |
| **扩展历史窗口** | CronusVLA, HAMLET | 固定视界限制，超出窗口的历史被丢弃 |
| **稀疏历史表示** | TraceVLA, HistRISE, Bpp | 省略关键细节，依赖外部模型稳定性 |
| **简单循环机制** | RoboFlamingo, AVA-VLA | 短窗口截断BPTT限制，无法学习长程依赖 |

## 🔬 核心方法

### 3.1 问题建模

**标准VLA（马尔可夫式）**:
```
A_{t:t+k} = π_θ(o_t, ℓ)
```
仅基于当前观察o_t和语言指令ℓ预测未来k步动作。

**ReMem-VLA（记忆感知式）**:
```
A_{t:t+k}, ô_{t-m} = π_θ(o_t, ℓ, h_t)
```
引入循环记忆状态h_t，累积历史上下文。

其中h_t = (h_t^c, h_t^f)包含两个组件：
- **帧级记忆h_t^f**: 每时间步更新
  ```
  h_t^f = F_f(h_{t-1}^f, o_{t-1}, ℓ)
  ```
- **块级记忆h_t^c**: 在块边界更新（每k帧）
  ```
  h_t^c = F_c(h_{t-k}^c, o_{t-k}, ℓ, h_{t-k}^f)
  ```

### 3.2 模型架构

ReMem-VLA包含五个关键组件：

#### 1. 冻结的VLM主干
- 采用 **Qwen3-VL-2B**（20亿参数视觉-语言模型）
- 处理多模态输入：RGB观察 o_t ∈ R^{V×H×W×3} 和语言指令 ℓ
- 输出隐藏状态 H_t ∈ R^{L×D}

#### 2. 任务特定的可学习查询
- **动作查询 Q^{action}** ∈ R^{N_a×D}: 提取动作生成关键特征
- **后见查询 Q^{img}** ∈ R^{N_img×D}: 提取过去视觉观察预测特征

#### 3. 双层循环记忆查询

**帧级记忆查询 Q^f** ∈ R^{N_f×D} - 短期记忆:
```
Q_t^f = β_f · Q̃_{t-1}^f + (1 - β_f) · Q_{t-1}^f
```
- 每帧密集更新
- 细粒度短期记忆，跟踪近期物体状态、正在进行的动作

**块级记忆查询 Q^c** ∈ R^{N_c×D} - 长期记忆:
```
Q_t^c = {
    β_c · Q̃_{t-k}^c + (1 - β_c) · Q_{t-k}^c,  if t mod k = 0
    Q_{t-1}^c,                                   otherwise
}
```
- 每k帧稀疏更新
- 较慢的指数衰减 (1-β_c)^{n/k} vs (1-β_f)^n
- 稳定的长期记忆，保持初始物体配置、任务进度

**关键设计 - 无梯度循环更新路径F**:
- VLM主干 + EMA融合机制
- 训练使用**截断BPTT**（窗口T=1）
- 记忆传播是确定性的前向更新，**保证记忆传播**
- 学习只关注记忆查询**写什么**，而非**如何传播**

#### 4. 基于Transformer的连接器
- 12层Transformer，双向自注意力
- 四种查询类型交互：Q^{action}, Q^{img}, Q^f, Q^c
- 允许动作和后见查询整合当前帧特征与累积时间上下文

#### 5. 预测头

**动作头** - 扩散模型:
```
L_action = E_{τ,ε~N(0,I)}[||ε - ε_θ(A_τ, τ, Q^{action})||^2]
```
- 预测动作块 A_{t:t+k} ∈ R^{k×7}
- 使用DDPM训练，DDIM推理（20步去噪）

**图像头** - 轻量级ViT风格解码器:
```
L_image = ||o_{t-m} - ô_{t-m}||_2^2
```
- 重构过去观察 o_{t-m}
- MSE损失监督

**总训练目标**:
```
L_total = L_action + λ_img · L_image
```
其中λ_img = 0.5

### 3.3 训练和推理

#### 基于槽的流式训练
**挑战**: 处理变长 episode 的循环机器人策略训练，必须严格按时间顺序处理，避免跨 episode 污染。

**传统方法**: 基于块的分批 → 切断时间连续性，丢弃长程上下文

**ReMem-VLA方案**: 槽式流式训练
- 维护固定池的B个并发槽，每个跟踪不同 episode 的当前时间步
- 每训练步从每个槽按序采样一帧，生成批次大小B
- 槽内循环状态在完整 episode 持续时间内保持和累积
- episode 边界硬重置槽的循环状态，严格隔离防止跨 episode 泄漏

#### 推理
- 每时间步维护两组循环查询 token
- 帧级查询每步刷新，块级查询以较粗频率更新
- 仅在新 episode 开始时重置
- **推理频率**: 单张RTX 4090 GPU上约**10 Hz**
- 有效控制率可通过重用预测动作块提升至近**20 Hz**

## 🧪 实验结果

### 4.1 模拟实验 - MemoryBench

**测试任务**（如图3所示）：

| 任务 | 描述 | 记忆需求 |
|------|------|----------|
| **Put Block Back** | 将块放到中心，按按钮，返回原位置 | 空间记忆 |
| **Rearrange Block** | 将块从中心移到红色区域，按按钮，再移回 | 空间+逻辑推理 |
| **Reopen Drawer** | 关闭抽屉，按按钮，再打开同一抽屉 | 空间记忆 |
| **Long Horizon Task** | Put Block Back + Rearrange Block组合（>600帧） | 长程空间+顺序记忆 |

**实验设置**:
- 每个任务100个训练演示
- 联合训练150k步
- 每任务100次 rollout 评估

**结果**（表1）：

| 方法 | Put Block Back | Rearrange Block | Reopen Drawer | Long Horizon | **平均** |
|------|----------------|-----------------|---------------|--------------|----------|
| OpenVLA-OFT | 0% | 0% | 3% | 0% | **0.75%** |
| π₀.₅ | 6% | 4% | 20% | 3% | **8.25%** |
| MemoryVLA | 0% | 1% | 5% | 0% | **1.5%** |
| **ReMem-VLA** | **93%** | **99%** | **100%** | **86%** | **94.5%** |

**关键发现**:
- ReMem-VLA在所有任务上显著超越所有基线
- 相比π₀.₅提升**86.25个百分点**
- 相比MemoryVLA提升**93个百分点**
- 在长程任务上保持86%成功率，展示强大长期记忆

### 4.2 真实世界实验

**任务设计**（评估四种记忆维度）：

| 任务 | 记忆类型 | 描述 |
|------|----------|------|
| **Water Flower** | 时间记忆 | 保持浇水姿势约6秒，无明显视觉反馈 |
| **Scoop Two Spoons** | 情景记忆 | 精确舀两勺米到锅，然后停止 |
| **Press Buttons** | 顺序+时间记忆 | 按绿-红-绿按钮，每个约3秒 |
| **Put Fruit Back** | 视觉记忆 | 将水果放回最初所在的盘子 |

**实验设置**:
- UR5机械臂 + Robotiq夹爪
- RealSense D435相机（腕部+第三视角）
- 每任务200个演示
- 每任务50次试验评估

**结果**（如图4所示）：

| 方法 | Water Flower | Scoop Rice | Press Buttons | Return Fruit | **平均** |
|------|--------------|------------|---------------|--------------|----------|
| π₀.₅ | ~11% | ~11% | ~11% | ~11% | **~11%** |
| MemoryVLA | ~8% | ~8% | ~8% | ~8% | **~8%** |
| **ReMem-VLA** | **高成功率** | **高成功率** | **高成功率** | **82%** | **82.5%** |

**关键发现**:
- ReMem-VLA平均成功率**82.5%**
- 远超MemoryVLA（8%）和π₀.₅（11%）
- 在视觉记忆任务（Return Fruit）上从34%提升至82%

### 4.3 消融研究

#### Q1: 各级循环查询的贡献？

**配置对比**（表2）：

| 配置 | Put Block | Rearrange | Reopen | Long Horizon | 平均 |
|------|-----------|-----------|--------|--------------|------|
| 无循环查询 | 7% | 32% | 27% | 5% | 17.75% |
| 仅帧级 | 90% | 95% | 96% | 70% | 87.75% |
| 仅块级 | 81% | 92% | 94% | 71% | 84.5% |
| **双级（本文）** | **93%** | **99%** | **100%** | **86%** | **94.5%** |

**结论**:
- 无循环查询 → 记忆能力完全崩溃
- 仅帧级 → 长期记忆不足
- 仅块级 → 短期记忆不足（按按钮需要）
- **双级配置 → 互补优势，最佳性能**

#### Q2: 过去观察预测（POP）的有效性？

| 任务 | 无POP | 有POP | 提升 |
|------|-------|-------|------|
| Water Flower | - | - | 有限 |
| Scoop Rice | - | - | 有限 |
| Press Buttons | - | - | 有限 |
| **Return Fruit** | **34%** | **82%** | **+48%** |

**结论**: POP在视觉记忆密集型任务上提供显著提升，双级循环查询在其他记忆维度强，但保留视觉细节较弱，需POP补充。

#### Q3: 无梯度vs可训练循环路径？

| 配置 | Put Block Back成功率 | 记忆能力 |
|------|----------------------|----------|
| 冻结VLM + EMA（本文） | **高** | **强** |
| 可训练VLM | 低 | 几乎消除 |
| 可训练GRU | 低 | 几乎消除 |
| 可训练MLP | 低 | 几乎消除 |

**结论**: 引入可训练参数到循环更新路径几乎完全消除模型记忆能力。

#### Q4: EMA超参数影响？

β参数扫描（0, 0.3, 0.5, 0.7, 0.9, 1）：
- **最佳**: β = 0.5
- 较大β → 过于激进更新，快速覆盖有用历史
- 较小β → 记忆过于惰性，难以吸收新信息

#### Q5: 循环记忆查询数量？

N ∈ {4, 16, 32, 64, 128, 256, 512}:
- **最佳**: N = 128
- 太少 → 记忆容量不足
- 太多 → 冗余增加注意力噪声，优化更难

#### Q6: 块级更新间隔？

测试0.5×, 1×, 2×, 3×动作块大小（30帧）：
- **最佳**: 1×动作块大小
- 平衡块级记忆的稳定性和新鲜度

## 💡 创新贡献

1. **双级循环记忆机制**
   - 帧级循环提供短期记忆
   - 块级循环支持长程保持
   - 首次全面覆盖空间、时间、视觉、情景、顺序记忆

2. **无梯度循环更新路径**
   - 冻结VLM + 固定EMA
   - 保证记忆传播，学习只关注"写什么"
   - 解决截断BPTT的长程依赖学习瓶颈

3. **过去观察预测策略**
   - 辅助训练目标增强视觉记忆
   - 重构先前RGB图像
   - 特别提升视觉记忆密集型任务

4. **槽式流式训练范式**
   - 支持变长 episode 的高效批训练
   - 保持时间连续性
   - 防止跨 episode 信息泄漏

5. **广泛的实验验证**
   - 模拟和真实世界双重验证
   - 覆盖多种记忆维度
   - 详尽消融研究提供实证洞察

## 📝 总结

ReMem-VLA通过创新的**双层循环记忆查询**机制，成功解决了VLA模型在记忆依赖任务上的系统性失败问题。其核心创新在于：

1. **帧级查询**提供细粒度短期记忆
2. **块级查询**提供稳定长期记忆
3. **无梯度更新路径**保证记忆有效传播
4. **过去观察预测**增强视觉记忆

实验结果表明，ReMem-VLA在模拟和真实世界环境中均展现出强大的多维度记忆能力，平均成功率相比基线提升超过**70个百分点**，为VLA模型在实际机器人应用中的可靠性奠定了重要基础。

**关键词**: Vision-Language-Action, Recurrent Memory, Short-term Memory, Long-term Memory, Robot Manipulation, Dual-Level Queries, Past Observation Prediction
