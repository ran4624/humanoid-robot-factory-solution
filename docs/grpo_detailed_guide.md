# GRPO (Group Relative Policy Optimization) 详解

> 原理、推导与应用效果全面分析

---

## 1. 什么是 GRPO？

**GRPO (Group Relative Policy Optimization)** 是 DeepSeek-AI 在 **DeepSeek-R1** 论文中提出的一种强化学习算法，专门用于大语言模型（LLM）的后训练（post-training）。

### 核心定位

| 特性 | 说明 |
|------|------|
| **全称** | Group Relative Policy Optimization |
| **提出者** | DeepSeek-AI (2025年1月) |
| **起源论文** | DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via Reinforcement Learning |
| **核心思想** | 通过组内相对奖励评估，消除对价值函数（Critic）的依赖 |
| **主要优势** | 内存效率、训练稳定、无需额外价值网络 |

---

## 2. 为什么需要 GRPO？

### 2.1 传统 RLHF/PPO 的问题

```
传统 PPO 的问题:
┌─────────────────────────────────────────────────────────┐
│ 1. 需要训练 Critic (价值网络)                             │
│    - 额外内存开销                                          │
│    - 增加训练复杂度                                        │
│    - 价值估计不准影响策略更新                              │
│                                                          │
│ 2. 优势估计方差大                                         │
│    - 单样本估计不可靠                                      │
│    - 需要复杂的 GAE 计算                                   │
│                                                          │
│ 3. 样本效率低                                             │
│    - 需要大量在线交互                                      │
│    - 在机器人/ reasoning 任务中成本高昂                    │
└─────────────────────────────────────────────────────────┘
```

### 2.2 GRPO 的创新点

**关键洞察**: 与其训练一个价值网络来估计每个状态的价值，不如**对同一个问题采样多个回答，用组内相对表现来估计优势**。

```
GRPO 的核心思想:
┌─────────────────────────────────────────────────────────┐
│                                                          │
│  问题: "解这个方程 2x + 5 = 13"                           │
│                                                          │
│  ┌─────────────────────────────────────────────────┐    │
│  │  采样 K=4 个回答                                 │    │
│  │  • 回答1: x=3, 正确 ✓ (奖励=1.0)                │    │
│  │  • 回答2: x=4, 错误 ✗ (奖励=0.0)                │    │
│  │  • 回答3: x=3, 正确 ✓ (奖励=1.0)                │    │
│  │  • 回答4: x=2, 错误 ✗ (奖励=0.0)                │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  组平均奖励 = 0.5                                         │
│                                                          │
│  相对优势:                                               │
│  • 回答1 优势 = 1.0 - 0.5 = +0.5 (优于平均水平)          │
│  • 回答2 优势 = 0.0 - 0.5 = -0.5 (差于平均水平)          │
│                                                          │
│  → 增加回答1的概率，减少回答2的概率                       │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## 3. GRPO 数学原理

### 3.1 目标函数

GRPO 的目标函数如下：

```
J_GRPO(θ) = E[q~P(Q), {o_i}_{i=1}^G ~ π_θ_old(O|q)]
            [1/G Σ_{i=1}^G ( min( r_i(θ) * A_i, 
                                 clip(r_i(θ), 1-ε, 1+ε) * A_i )
                         - β * KL(π_θ || π_ref) )]
```

其中：
- `q`: 查询/问题
- `G`: 每组采样的回答数量（Group Size）
- `o_i`: 第 i 个回答
- `r_i(θ) = π_θ(o_i|q) / π_θ_old(o_i|q)`: 重要性采样比率
- `A_i`: 回答 i 的相对优势
- `ε`: PPO 裁剪参数（通常 0.2）
- `β`: KL 惩罚系数

### 3.2 相对优势计算（核心创新）

**无需 Critic，直接用组内奖励归一化：**

```
A_i = (r_i - mean({r_1, r_2, ..., r_G})) / std({r_1, r_2, ..., r_G})
```

或者简化版本（仅减去均值）：

```
A_i = r_i - mean({r_1, r_2, ..., r_G})
```

**在 AtomVLA 中的形式：**

```python
# 对每个状态采样 K 个候选动作
K = 10
candidates = [policy.sample() for _ in range(K)]

# 计算每个候选的奖励
rewards = [reward_model(c) for c in candidates]

# 计算相对优势（归一化）
mean_reward = np.mean(rewards)
std_reward = np.std(rewards) + 1e-8
advantages = [(r - mean_reward) / std_reward for r in rewards]

# GRPO 损失
loss = -sum(advantage * log_prob for advantage, log_prob in zip(advantages, log_probs))
```

### 3.3 完整训练流程

```
Algorithm: GRPO
─────────────────────────────────────────────────────────
输入: 预训练模型 π_ref, 奖励模型 R, 数据集 D
超参数: G (组大小), ε (裁剪阈值), β (KL系数), η (学习率)

1. 初始化策略 π_θ ← π_ref
2. for iteration = 1, 2, ... do
3.    从 D 中采样 batch 的问题 {q}
4.    对于每个问题 q:
5.       使用当前策略 π_θ 采样 G 个回答 {o_1, ..., o_G}
6.       计算奖励 {r_1, ..., r_G} = R(q, o_i)
7.       计算相对优势 A_i = (r_i - mean(r)) / std(r)
8.    计算 GRPO 损失 L_GRPO(θ)
9.    更新策略: θ ← θ - η * ∇_θ L_GRPO(θ)
10. end for
─────────────────────────────────────────────────────────
```

## 4. GRPO 的关键优势

### 4.1 内存效率

```
对比 (对于 7B 模型):
┌────────────────────────────────────────────────────────┐
│ PPO 内存占用:                                            │
│ - 策略网络 (Policy): ~14 GB                              │
│ - 价值网络 (Critic): ~14 GB                              │
│ - 参考模型 (Reference): ~14 GB                           │
│ - 总计: ~42 GB                                          │
├────────────────────────────────────────────────────────┤
│ GRPO 内存占用:                                           │
│ - 策略网络 (Policy): ~14 GB                              │
│ - 参考模型 (Reference): ~14 GB                           │
│ - 总计: ~28 GB                                          │
├────────────────────────────────────────────────────────┤
│ 节省: ~33% 显存！                                       │
└────────────────────────────────────────────────────────┘
```

### 4.2 训练稳定性

| 特性 | PPO | GRPO |
|------|-----|------|
| 价值估计误差 | 高 | 无 (不需要) |
| 优势估计方差 | 大 | 小 (组内归一化) |
| 训练崩溃风险 | 较高 | 较低 |
| 超参数敏感度 | 高 | 中等 |

### 4.3 适用于可验证奖励任务

GRPO 特别适合奖励明确可验证的任务：

- ✅ 数学题（答案正确/错误）
- ✅ 代码生成（测试通过/失败）
- ✅ 机器人操作（任务成功/失败）
- ✅ 逻辑推理（结果验证）

## 5. GRPO 在 AtomVLA 中的应用

### 5.1 具体实现

**在 AtomVLA 中的 GRPO 变体：**

```python
# 损失函数 (来自 AtomVLA 论文)
L_grpo = -E[1/K Σ_k (A^(k) log π_φ(ã_{t:t+N}^(k)|H_t) 
                      - λ_KL KL(π_φ || π_ref))]
```

**奖励组成：**

```
r^(k) = -(λ_sub * E_sub^(k) + λ_goal * E_goal^(k) + α * D^(k))

其中:
- E_sub: 子目标能量 (V-JEPA2 预测 vs 子目标)
- E_goal: 最终目标能量 (V-JEPA2 预测 vs 最终目标)  
- D: 模仿偏差 (与 SFT 行为的距离)
- λ_sub=0.3, λ_goal=0.4, α=0.3
```

### 5.2 为什么 AtomVLA 选择 GRPO？

| 原因 | 说明 |
|------|------|
| **离线训练** | 无需昂贵的真实机器人交互 |
| **多候选评估** | 每个状态采样10个动作块，选择最优 |
| **世界模型集成** | 使用 V-JEPA2 评估候选动作 |
| **稳定训练** | KL 约束保持与 SFT 策略的稳定性 |

### 5.3 性能提升

```

│ SFT + GRPO:      97.0% 成功率                          │
│                  ↑ 4.0% 提升                           │
├────────────────────────────────────────────────────────┤
```

## 7. GRPO 变体与改进

### 7.1 主要变体

```
GRPO 变体家族:
┌─────────────────────────────────────────────────────────┐
│                                                          │
│  标准 GRPO (DeepSeek-R1)                                │
│  └── 组内奖励归一化优势估计                              │

│      (难度自适应优势计算)                                │
│                                                          │
│  Graph-GRPO                                             │
│  └── 用于多智能体拓扑学习                               │
│                                                          │
│  Causal-GRPO                                            │
│  └── 因果感知的 GRPO 变体                               │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 7.2 超参数调优建议

| 超参数 | 推荐值 | 说明 |
|--------|--------|------|
| 组大小 (G) | 4-16 | 越大估计越准，但计算成本增加 |
| 裁剪阈值 (ε) | 0.2 | 标准 PPO 值 |
| KL 系数 (β) | 0.01-0.1 | 防止偏离参考策略太远 |
| 学习率 (η) | 1e-6 to 1e-5 | 比 SFT 更小 |
| 温度 (T) | 0.5-1.0 | 控制采样多样性 |

## 8. GRPO 的局限性与注意事项

### 8.1 局限性

1. **奖励可验证性要求**
   - 需要明确可验证的奖励信号
   - 不适用于主观评估任务（如创意写作）

2. **组大小权衡**
   - 小组：方差大，估计不准
   - 大组：计算成本高
   - 通常 G=4-16 是平衡点

3. **稀疏奖励问题**
   - 在奖励极其稀疏的场景中，组内可能全为0或1
   - 优势估计失效

### 8.2 最佳实践

```
✅ 适用场景:
   - 数学问题求解 (答案可验证)
   - 代码生成 (测试用例通过)
   - 机器人操作 (任务成功/失败)
   - 逻辑推理 (结果正确性可验证)

❌ 不适用场景:
   - 开放式生成 (创意写作、对话)
   - 主观质量评估 (艺术性、风格)
   - 需要细粒度反馈的任务
```

## 9. 总结

### GRPO 核心要点

| 维度 | 内容 |
|------|------|
| **核心创新** | 用组内相对奖励替代价值网络 |
| **主要优势** | 省显存、训练稳定、适合可验证任务 |
| **关键公式** | `A_i = (r_i - mean(r)) / std(r)` |
| **最佳应用** | Math, Code, Robotics, Reasoning |
| **代表工作** | DeepSeek-R1, AtomVLA, Flow-GRPO |

### 为什么 GRPO 如此有效？

1. **消除 Critic**: 节省 30%+ 显存
2. **组内归一化**: 自动适应奖励尺度
3. **多候选比较**: 比单样本估计更可靠
4. **即插即用**: 易于集成到现有 RLHF 流程

### 未来方向

- 自适应组大小
- 分层 GRPO (Hierarchical GRPO)
- 与在线 RL 结合
- 应用到更多模态 (Video, 3D, Audio)
