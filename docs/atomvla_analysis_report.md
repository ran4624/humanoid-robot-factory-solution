## 执行摘要

AtomVLA提出了一种**可扩展的VLA后训练框架**，通过结合**LLM子任务分解**和**预测性潜在世界模型**，解决了长时程机器人任务中的复合错误问题。

### 核心创新

| 创新点 | 描述 |
|--------|------|
| **子任务感知架构** | 使用GPT-4o将高层指令分解为细粒度原子子任务 |
| **潜在世界模型** | 基于V-JEPA2在潜在空间评估候选动作轨迹 |
| **离线GRPO训练** | 无需昂贵在线交互的可扩展强化学习后训练 |
| **动作块生成** | 使用Flow-Matching Diffusion Transformer生成动作序列 |

### 主要成果

- ✅ **LIBERO基准**: 97.0% 成功率（SOTA）
- ✅ **LIBERO-PRO基准**: 48.0% 成功率（强泛化能力）
- ✅ **真实世界验证**: Galaxea R1 Lite平台，成功完成T恤折叠等复杂任务
- ✅ **模型规模**: 仅4B参数，无需跨形态大规模预训练

## 核心问题

### 1.1 当前VLA模型的局限性

当前VLA（Vision-Language-Action）模型面临三大挑战：

```
挑战1: 计算资源需求巨大
    ├── 需要大规模跨形态预训练
    ├── 难以在资源受限环境部署
    └── 动作空间差异导致不一致收益

挑战2: 语义-视觉信息不对称
    ├── 高维视觉token主导潜在空间
    ├── 稀疏语言指令难以落地为物理指导
    └── 长时程任务中复合错误严重

挑战3: 在线强化学习成本高昂
    ├── 真实世界交互成本极高
    ├── 安全风险限制在线探索
    └── 静态模仿学习与动态策略优化之间存在鸿沟
```

### 1.2 指令落地差距（Instruction Grounding Gap）

**核心问题**: 当前范式主要依赖粗粒度、高层任务指令进行监督微调，缺乏明确的中间指导，导致长时程任务中严重的复合错误。

**AtomVLA的解决方案**:
- 使用LLM将复杂高层演示分解为细粒度**原子子任务**
- 提供明确的阶段性指导，对齐策略的语义理解与动作块机制
- 利用世界模型在潜在空间评估候选动作轨迹

```

├─────────────────────────────────────────────────────────────────────┤
│                                                                      │

│  ┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐  │
│  │  高层指令        │───→│  LLM分解        │───→│  原子子任务指令  │  │
│  │  (High-level)   │    │  (GPT-4o)       │    │  (Atomic)       │  │
│  └─────────────────┘    └─────────────────┘    └─────────────────┘  │
│                                                        │            │
│  ┌─────────────────────────────────────────────────────┐            │
│  │  Qwen3-VL-4B + Flow-Matching Action Head            │            │
│  │  • 多视角视觉观测 Ot                                │            │
│  │  • 高层指令 It + 子任务指令 SI_t                    │            │

│  │  • 编码当前观测 J(O_t)                                       │    │
│  │  • 预测候选动作的未来状态 W_θ(J(O_t), ã_{t:t+N})            │    │
│  │  • 在潜在空间评估与目标的一致性                              │    │
```

### 2.2 模型架构详解

#### 2.2.1 VLM主干网络

- **基础模型**: Qwen3-VL-4B-Instruct
- **输入**:
  - 多视角视觉观测 O_t（第三人称相机 + 腕部相机）
  - 高层指令 I_t
  - 子任务指令 SI_t
- **输出**: 上下文token特征 H_t = f_VLM(O_t, I_t + SI_t)

#### 2.2.2 动作头（Action Head）

**架构**: 交叉注意力Diffusion Transformer (DiT)

**Flow-Matching公式**:
```
1. 添加噪声: x_τ = (1-τ)ε + τ·a_{t:t+N}, 其中 ε ~ N(0,I), τ ∈ (0,1)
2. 预测目标速度: v = a_{t:t+N} - ε
3. 训练目标: L_FM = E[||A_φ(x_τ, τ; H_t, s_t) - (a_{t:t+N} - ε)||²]
4. 推理: 从a^(0) ~ N(0,I)开始，迭代K步细化
   a^(k+1) = a^(k) + Δt·A_φ(a^(k), τ_k; H_t, s_t)
```

#### 2.2.3 子任务数据集构建

**使用GPT-4o进行子任务分解**:

```json
{
  "tasks": [
    {
      "instruction": "Pick up the red cola can",
      "start_frame": 0,
      "end_frame": 45
    },
    {
      "instruction": "Place the red cola can into the white basket",
      "start_frame": 46,
      "end_frame": 120
    }
  ]
}
```

**标准化词汇表**:
- `Pick up [object]`
- `Place [object] into/on [target]`
- `Move arm to [location]`
- `Open/Close [object]`
- `Push [object]`

**约束**: 每个任务分解为2-5个子任务，与基本操作动作对齐

## 两阶段训练流程

### 3.1 Stage I: 监督微调 (SFT)

**训练配置**:
| 超参数 | 值 |
|--------|-----|
| GPU | 4× NVIDIA H100 (80GB) |
| 学习率 | 5e-4 |
| Batch Size | 8 (per GPU) |
| Epoch | 2 |
| 输入图像 | 1×第三人称 + 1×腕部相机 (224×224) |
| 动作块大小 | 4步 |
| 动作维度 | 7 DoF |

**损失函数**: Flow-Matching Loss

**关键发现**: 即使没有跨形态大规模预训练，仅在领域内多任务演示上训练也能达到93%成功率

### 3.2 Stage II: 强化学习后训练

#### 3.2.1 动作条件世界模型

**基础**: V-JEPA2 (Video Joint Embedding Predictive Architecture)

**流程**:
```
1. 冻结视觉编码器 J(·): 图像 → 潜在token序列
2. 预测器网络 W_θ: 基于当前观测J(O_t)和候选动作ã_{t:t+N}预测未来状态ẑ_{t+N}
3. 在潜在空间比较预测结果与目标状态
```

#### 3.2.2 奖励设计

**子目标能量**:
```
E_sub^(k) = ||W_θ(J(O_t), ã_{t:t+N}^(k)) - J(O_b(t))||_1
```

**最终目标能量**:
```
E_goal^(k) = ||W_θ(J(O_t), ã_{t:t+N}^(k)) - J(O_{T-1})||_1
```

**模仿偏差**:
```
D^(k) = ||ã_{t:t+N}^(k) - a*_{t:t+N}||_1
```

**最终奖励**:
```
r^(k) = -(λ_sub·E_sub^(k) + λ_goal·E_goal^(k) + α·D^(k))

其中: λ_sub = 0.3, λ_goal = 0.4, α = 0.3
```

#### 3.2.3 GRPO优化

**Group Relative Policy Optimization**:

```
L_GRPO = -E[1/K Σ_k (A^(k) log π_φ(ã_{t:t+N}^(k)|H_t) - λ_KL KL(π_φ || π_ref))]
```

**优势计算**: 对每个状态的K个候选进行奖励归一化

**训练配置**:
| 超参数 | 值 |
|--------|-----|
| GPU | 2× NVIDIA H100 (80GB) |
| Batch Size | 8 (per GPU) |
| 采样轨迹数 | 100K |
| 候选数 (K) | 10 |
| KL惩罚系数 (λ) | 0.01 |

**关键设计**: 仅更新动作头参数，保持VLM主干稳定

## 关键发现与洞察

### 5.1 核心洞察

#### 洞察1: 跨形态预训练并非必需

**发现**: 即使没有跨形态大规模预训练，仅在领域内多任务演示上训练也能达到强性能。

**启示**: 
- 训练成本显著降低
- 避免了不同机器人动作空间的差异问题
- 数据质量比数据规模更重要

#### 洞察2: 后训练持续改进性能

**发现**: GRPO后训练在所有LIBERO套件上都带来提升，在LIBERO-Goal上提升最大(+6.0%)。

**原因**:
- SFT提供强模仿先验
- GRPO通过群体级比较监督改进长时程目标对齐
- 世界模型评估强化与目标一致的轨迹

#### 洞察3: 子任务指令提供显式指导

**发现**: 添加原子子任务指令在LIBERO-Long上额外提升2.2%。

**机制**:
- 提供当前阶段的明确中间目标
- 保留整体任务目标
- 改善长时程任务中的信用分配

#### 洞察4: 潜在世界模型避免像素级幻觉

**优势**:
- 在潜在空间评估避免像素级合成的视觉伪影
- 自回归错误不会累积
- 计算开销显著降低

### 5.2 设计选择分析

#### 5.2.1 为什么使用V-JEPA2?

**V-JEPA2优势**:
- 自监督预训练，无需动作标注
- 学习有意义的视觉表示
- 预测未来状态而非生成像素
- 计算效率高

#### 5.2.2 为什么使用Flow-Matching?

**对比其他动作表示**:
- **离散token** (如FAST): 量化误差，信息损失
- **连续回归** (如L1): 模式平均，多模态问题
- **Flow-Matching**: 
  - 支持多模态动作分布
  - 稳定训练
  - 高质量动作生成

#### 5.2.3 为什么离线GRPO?

**对比在线RL**:
- 无需昂贵的真实世界交互
- 避免安全风险
- 可扩展性强
- 利用已有演示数据

## 局限性与未来方向

### 7.1 当前局限

1. **静态子任务边界**
   - 子任务边界由LLM预先确定
   - 无法适应高度动态环境
   - 缺乏执行时的灵活性

2. **世界模型不确定性**
   - 当前系统无法感知预测不确定性
   - 在复杂场景中可能产生不准确预测

3. **未见对象泛化**
   - 对完全未见过的对象泛化有限
   - 新语义指令适应性待提升

### 7.2 未来工作

1. **端到端子任务生成**
   - 执行时动态生成子任务
   - 根据环境反馈调整子任务边界

2. **不确定性感知世界模型**
   - 显式建模预测不确定性
   - 基于不确定性的自适应规划

3. **增强泛化能力**
   - 处理完全未见对象
   - 适应新颖语义指令

4. **更高效的训练**
   - 减少对世界模型的依赖
   - 探索无模型RL方法

### 核心贡献总结
